Process Automation eXtensible Markup Language.

Introduction

Paxml is an XML-based scripting language, providing simple yet powerful syntax to automate your business operations that you would otherwise have to manually perform. For instance, accessing database and files, driving browsers, calling web APIs and glue all these together into your operational work flow. Most importantly it automates the repetitive part for you, drastically increasing the efficiency of your work and liberating you to do more creative work.

Therefore, Paxml is suitable for operational staff to automate business processes and for software testers to automate test executions.

Paxml is also an open source software released under Affero General Public License Version 3.0.

Taste it a bit

Here is a simple example. Suppose you have an excel file which contains 1000 rows of English terms in column A that you want to let Google translate into German and save the translation back to column B. You can simply use the XML below to have the job done for you, iterating through all those 1000 rows and for each term open web browser to let Google translate do the work:

<scenario>
  <readExcel id="excel" file="MyExcel.xls" />
  <iterate values="${excel}">
    <url>https://translate.google.com/#en/de</url>
    <type locator="source">${var.A}</type>
    <click locator="gt-lang-submit"/>
    <writeExcel file="${excel}" row="${index+1}" column="B" >    
      <pick locator="result_box" timeout="5000"/>
    </writeExcel>
  </iterate>
</scenario>

Save the above XML as file named TranslateExcel.xml then run this command to let it run:

paxml TranslateExcel

Now you can start using column B which contains 1000 rows of German translation!

What about software tester?

Suppose you are a tester hired by Google to test their translation website, and you use the same excel file to store your test data where column C stores the correct German translation. You need to verify for all those 1000 translations by comparing each row's column B and column C. Then you just need to create a new XML file like below, to first call the above XML file to acquire the translations, then compare column B and C for each row:

<scenario>
  <TranslateExcel/>
  <readExcel id="excel" file="MyExcel.xls"/>
  <iterate values="${excel}" >
    <assert expected="${var.C}" actual="${var.B}" />
  </iterate>
</scenario>

And save this file as TestTranslate.xml then run this command to run the test:

paxml TestTranslate
